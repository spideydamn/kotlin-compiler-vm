# Спецификация семантики языка

## Приоритеты и ассоциативность

Операторы имеют стандартный приоритет и левоассоциативность:
- `* / %` выше `+ -`
- Сравнения выше логических
- `&&` выше `||`
- Унарные операторы (`!`, `-`, `+`) применяются к следующему выражению

## Арифметические операции и типы

Разрешены только операции над операндами одного типа.

**Допустимые:**
- `int ∘ int → int` (где ∘ ∈ {+, -, *, /, %})
- `float ∘ float → float` (где ∘ ∈ {+, -, *, /})
- `int % int → int` (только для остатка от деления)

**Недопустимые:**
- `int ∘ float` - ошибка компиляции
- `float ∘ int` - ошибка компиляции
- `float % float` - ошибка компиляции

## Перекрытие идентификаторов (Shadowing)

Идентификатор может быть повторно объявлен во вложенной области видимости. Новая декларация временно скрывает предыдущее определение до выхода из текущей области.

**Разрешено:**
```kotlin
let x: int = 10;
{
    let x: int = 20;  // перекрывает внешнюю x
}
```

**Запрещено:**
```kotlin
let x: int = 10;
let x: int = 20;  // ошибка: дублирование в одной области
```
