# Спецификация семантики языка программирования

## Ассоциативность и приоритеты

Операторы имеют стандартный приоритет и левоассоциативность:

* `* / %` выше `+ -`
* сравнения (`==`, `!=`, `<`, `>`, `<=`, `>=`) — выше логических
* `&&` выше `||`
* унарные операторы (`!`, `-`, `+`) применяются к следующему выражению

## Приведение типов для арифметических операций

Арифметические операции `+`, `-`, `*`, `/`, `%` определены для типов `int` и `float` со следующими правилами:

1. **int ∘ int → int**
   Результат — `int`.

2. **int ∘ float → float**
   Операнд `int` неявно приводится к `float`.
   Результат — `float`.

3. **float ∘ int → float**
   Операнд `int` неявно приводится к `float`.
   Результат — `float`.

4. **float ∘ float → float**
   Результат — `float`.

5. **Оператор %**

   * Допустим только для `int % int → int`.
   * Любые комбинации с `float` являются ошибкой.

## Перекрытие идентификаторов (Shadowing)

Идентификатор может быть повторно объявлен во вложенной области видимости. Новая декларация временно скрывает предыдущее определение до выхода из текущей области. Shadowing разрешено для переменных и параметров. Ошибкой является только повторное объявление идентификатора *в пределах одной и той же области видимости*.
